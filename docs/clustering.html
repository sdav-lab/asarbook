12.1 Learning Objectives

By the end of this chapter, you will be able to:

  1. Understand clustering concepts and when to use them
  2. Prepare data for clustering analysis (scaling and normalization)

  4. Determine the optimal number of clusters
  5. Visualize and interpret cluster results
  6. Use hierarchical clustering methods
  7. Apply clustering to player and team segmentation
  8. Leverage AI tools to enhance clustering analysis

12.2 Introduction

Clustering is an unsupervised learning technique that groups similar observations together without predefined labels. Unlike regression where we predict an outcome, clustering discovers hidden patterns and structures in data.

12.2.1 Why Use Clustering in Sport Analytics?

  • Player Segmentation: Group players by playing style (scorers, defenders, all-around)
  • Team Archetypes: Identify different types of teams (offense-oriented, defense-oriented, balanced)
  • Fan Segmentation: Cluster fans by behavior for targeted marketing
  • Opponent Analysis: Group opponents by strategy to prepare game plans
  • Performance Profiles: Identify different performance patterns over time

12.2.2 Types of Clustering

  1. K-means: Partition data into k distinct clusters (we’ll focus on this)
  2. Hierarchical: Build a tree of clusters
  3. DBSCAN: Density-based clustering
  4. PAM (Partitioning Around Medoids): More robust version of k-means

12.3 K-Means Clustering

K-means is the most popular clustering algorithm. It partitions data into k clusters by:

  1. Randomly selecting k initial cluster centers
  2. Assigning each point to the nearest center
  3. Recalculating centers as the mean of assigned points
  4. Repeating steps 2-3 until convergence

12.3.2 Why Scale Data?

Critical: Always scale data before clustering!

# Without scaling - variables with larger ranges dominate
range(players$points_per_game)
## [1]  1.7 30.6
range(players$steals_per_game)
## [1] 0 3
# Points vary 0-30, steals vary 0-3
# Distance calculations would be dominated by points!

# Example: Distance between two players
player1 <- c(points = 20, steals = 2.0)
player2 <- c(points = 18, steals = 1.0)

# Euclidean distance
dist_unscaled <- sqrt((20-18)^2 + (2.0-1.0)^2)
cat("Unscaled distance:", dist_unscaled, "\n")
## Unscaled distance: 2.236068
# Points contribute 2^2 = 4, steals contribute 1^2 = 1
# Points dominate!

# After scaling (z-scores)
player1_scaled <- c(points = 0, steals = 0)    # Both at mean
player2_scaled <- c(points = -0.5, steals = -0.8)
dist_scaled <- sqrt((-0.5)^2 + (-0.8)^2)
cat("Scaled distance:", dist_scaled, "\n")
## Scaled distance: 0.9433981
# Now both variables contribute equally

12.4 Determining Optimal Number of Clusters

Key Question: How many clusters should we use?

12.4.1 Method 1: Elbow Method (Within-Cluster Sum of Squares)

# Elbow method: look for "elbow" where adding clusters gives diminishing returns
fviz_nbclust(players_scaled, kmeans, method = "wss", k.max = 10) +
  labs(
    title = "Elbow Method: Optimal Number of Clusters",
    subtitle = "Look for the 'elbow' where the curve bends",
    x = "Number of Clusters (k)",
    y = "Total Within-Cluster Sum of Squares"
  ) +
  theme_minimal() +
  geom_vline(xintercept = 3, linetype = "dashed", color = "red")

Interpretation: The “elbow” is where adding more clusters doesn’t substantially reduce within-cluster variance. Often around k=3 or k=4 in this example.

12.4.2 Method 2: Silhouette Method

# Silhouette method: measures how well each point fits in its cluster
fviz_nbclust(players_scaled, kmeans, method = "silhouette", k.max = 10) +
  labs(
    title = "Silhouette Method: Optimal Number of Clusters",
    subtitle = "Higher average silhouette width = better clustering",
    x = "Number of Clusters (k)",
    y = "Average Silhouette Width"
  ) +
  theme_minimal()

Interpretation: Choose k with the highest average silhouette width. Values: - 0.71-1.0: Strong structure - 0.51-0.70: Reasonable structure - 0.26-0.50: Weak structure - < 0.25: No substantial structure

12.4.3 Method 3: Gap Statistic

# Gap statistic: compares within-cluster dispersion to null reference
set.seed(123)
gap_stat <- clusGap(players_scaled, FUN = kmeans, nstart = 25,
                    K.max = 10, B = 50)
## Clustering k = 1,2,..., K.max (= 10): .. done
## Bootstrapping, b = 1,2,..., B (= 50)  [one "." per sample]:
## .................................................. 50
fviz_gap_stat(gap_stat) +
  labs(title = "Gap Statistic Method") +
  theme_minimal()

# Optimal k
cat("Optimal k (gap statistic):", maxSE(gap_stat$Tab[, "gap"], gap_stat$Tab[, "SE.sim"]), "\n")

12.13 Best Practices

12.13.1 DO:

  • ✅ Always scale/standardize variables before clustering
  • ✅ Try multiple values of k and compare
  • ✅ Use multiple methods to determine optimal k
  • ✅ Validate clusters with silhouette analysis
  • ✅ Interpret clusters in domain context (sport analytics)
  • ✅ Consider clustering subgroups separately (by position, conference, etc.)

12.13.2 DON’T:

  • ❌ Include categorical variables directly (convert to dummy variables first)
  • ❌ Include highly correlated variables (causes redundancy)
  • ❌ Include ID variables or names
  • ❌ Forget to set seed for reproducibility
  • ❌ Trust a single clustering method without validation
  • ❌ Over-interpret small differences between clusters
